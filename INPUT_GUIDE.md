# 교회 자막 웹앱 — 사람이 이 앱에서 해야 할 일

이 앱에서 **사람이 할 일**은 다음 네 가지입니다.

---

## 1. 원고 DOCX를 첨부할 것

- **설교 원고 DOCX** 파일을 업로드합니다.
- Word에서 본문/인용 구절을 **빨간색(#FF0000)** 으로 표시해 두면, 앱이 이를 추출해 AI용 데이터로 씁니다.
- 파싱 결과(빨간 글씨만 모은 목록·전체 구조)를 보고, 필요하면 AI에 넣어 **자막용 코드**를 생성받습니다.

---

## 2. 필요하다면 찬송(hymn)을 업데이트/변경할 것

- 이번 주에 사용할 **찬송가 목록**을 앱에서 입력하거나 수정합니다.
- 한 줄에 한 곡씩 찬송가 제목을 적습니다. (예: `임재`, `좋으신 하나님`)
- `EvergreenSlideMaker/Hymn/hymn.txt` 에 해당 제목들이 있어야 슬라이드에 반영됩니다. 필요하면 그 파일을 직접 추가·수정합니다.

---

## 3. 필요하다면 다른 이미지칸·카드 페이지도 만들 수 있을 것

- **카드 슬라이드** (예: 통성기도, 대표기도, 성가대 찬양, 광고, 축도 등)를 추가할 수 있습니다.
- 앱에서 “카드 슬라이드 텍스트”를 입력하면, 해당 문구가 적힌 슬라이드가 순서대로 들어갑니다.
- 이미지 슬라이드가 필요하면, 기존처럼 `EvergreenSlideMaker/image/` 에 이미지를 넣고, 생성 코드에서 `add_image_slide` 를 사용하는 방식으로 구성할 수 있습니다.

---

## 4. 최종적으로 파일을 추출할 것

- 위에서 준비한 **원고 기반 자막 코드**와, **찬송 목록**, **카드 슬라이드**를 반영한 뒤 **「PPTX 생성 후 다운로드」** 를 눌러 최종 자막용 PPTX 파일을 받습니다.

---

## 서버 실행 조건

- **EvergreenSlideMaker** 폴더가 프로젝트 루트에 있어야 합니다.

### Vercel로 배포 (권장)

- 이 저장소를 **GitHub**에 push한 뒤 **Vercel**에서 해당 저장소를 연결해 배포하면 됩니다.
- 저장소에 파일(찬송가, 이미지 등)을 추가·수정하고 push하면 Vercel이 자동으로 재배포해 앱에 반영됩니다.
- 자세한 절차는 **[DEPLOY.md](./DEPLOY.md)** 를 참고하세요.

### 로컬 실행 (FastAPI)

- `webapp` 폴더 기준으로 실행:
  ```bash
  cd c:\Users\woorim\evergreen_church
  pip install -r webapp/requirements.txt
  uvicorn webapp.main:app --reload
  ```
- 브라우저에서 `http://127.0.0.1:8000` 접속 후, 1 → 2 → 3 → 4 순서로 진행하면 됩니다.
